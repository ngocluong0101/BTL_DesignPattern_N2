Tài Liệu Kỹ Thuật Dự Án

Ứng dụng Desktop quản lý bán quần áo - Nhóm 5 thành viên

Giới thiệu dự án

Dự án "Xây dựng ứng dụng desktop quản lý bán quần áo" là phần mềm quản lý toàn diện cho cửa hàng thời trang. Cơ sở dữ liệu được thiết kế tổng thể và thống nhất với 28 bảng chuẩn 3NF, phân bổ khéo léo để hỗ trợ độc lập cho 5 module do 5 thành viên phụ trách.


Công nghệ sử dụng: Java Core, Java Swing, MySQL (JDBC thuần không ORM), Kiến trúc 5-Tier.

Phân công nhiệm vụ nhóm (5 thành viên)

Thành viên Module phụ trách Design Pattern áp dụng
Ly Quản lý hóa đơn, Sự kiện, Thông báo Factory Method, Observer
Linh Quản lý sản phẩm, Loại, Kho Abstract Factory, Adapter
Lương Khách hàng, Tài khoản, Khuyến mãi Decorator, Builder
An Quản lý đặt hàng, Voucher, Tích điểm Strategy, State
Minh Đăng nhập, Nhà cung cấp, Checkout Singleton, Facade

Nhóm Tài khoản & Khách hàng Lương, Minh

1. Bảng `roles`

Tên cột Kiểu dữ liệu Ràng buộc
id INT PK AUTO_INCREMENT
role_name VARCHAR(50) UNIQUE, NOT NULL

2. Bảng `users` (Hỗ trợ Singleton)

Tên cột Kiểu dữ liệu Ràng buộc
id INT PK AUTO_INCREMENT
role_id INT FK NOT NULL
username VARCHAR(50) UNIQUE, NOT NULL
password VARCHAR(255) NOT NULL
full_name VARCHAR(100) NOT NULL
status BOOLEAN DEFAULT 1

3. Bảng `user_sessions`

Tên cột Kiểu dữ liệu Ràng buộc
id INT PK AUTO_INCREMENT
user_id INT FK NOT NULL
login_time DATETIME DEFAULT CURRENT_TIMESTAMP
token VARCHAR(255)

4. Bảng `membership_levels`

Tên cột Kiểu dữ liệu Ràng buộc
id INT PK AUTO_INCREMENT
level_name VARCHAR(50) NOT NULL
min_points INT NOT NULL

5. Bảng `customers` (Hỗ trợ Builder)

Tên cột Kiểu dữ liệu Ràng buộc
id INT PK AUTO_INCREMENT
user_id INT FK UNIQUE, Có thể NULL
membership_level_id INT FK
full_name VARCHAR(100) NOT NULL
phone VARCHAR(20) UNIQUE, NOT NULL
email VARCHAR(100)

Nhóm Nhà cung cấp Minh

6. Bảng `supplier_statuses`

Tên cột Kiểu dữ liệu Ràng buộc
id INT PK AUTO_INCREMENT
status_name VARCHAR(50) NOT NULL

7. Bảng `suppliers`

Tên cột Kiểu dữ liệu Ràng buộc
id INT PK AUTO_INCREMENT
status_id INT FK NOT NULL
name VARCHAR(100) NOT NULL
phone VARCHAR(20) NOT NULL

Nhóm Sản phẩm & Kho Linh

8. Bảng `categories` & 9. `product_statuses` & 10. `sizes` & 11. `colors`

Các bảng danh mục cơ bản (id INT PK, name/status_name VARCHAR NOT NULL).

12. Bảng `products` (Hỗ trợ Abstract Factory, Adapter)

Tên cột Kiểu dữ liệu Ràng buộc
id INT PK AUTO_INCREMENT
category_id INT FK NOT NULL
supplier_id INT FK NOT NULL
status_id INT FK NOT NULL
name VARCHAR(200) NOT NULL
product_type VARCHAR(50) NOT NULL (VD: Basic, Premium)

13. Bảng `product_variants` (Tồn kho thực tế)

Tên cột Kiểu dữ liệu Ràng buộc
id INT PK AUTO_INCREMENT
product_id INT FK NOT NULL
size_id INT FK NOT NULL
color_id INT FK NOT NULL
sku VARCHAR(50) UNIQUE, NOT NULL
price DECIMAL(12,2) NOT NULL
stock_quantity INT DEFAULT 0

Nhóm Khuyến mãi & Voucher Lương, An

14. Bảng `promotions` & 15. `product_promotions` (Lương - Hỗ trợ Decorator)

Tên cột Kiểu dữ liệu Ràng buộc
id INT PK AUTO_INCREMENT (promotions)
discount_type VARCHAR(20) NOT NULL (PERCENT, FIXED)
discount_value DECIMAL(12,2) NOT NULL
product_id, promotion_id INT PKFK (product_promotions)

16. Bảng `vouchers` (An - Hỗ trợ Strategy)

Tên cột Kiểu dữ liệu Ràng buộc
id INT PK AUTO_INCREMENT
code VARCHAR(50) UNIQUE, NOT NULL
discount_type VARCHAR(20) NOT NULL (PERCENT, FIXED, FREE_SHIP)
discount_value DECIMAL(12,2) NOT NULL
min_order_value DECIMAL(12,2) DEFAULT 0

Nhóm Đặt hàng & Tích điểm An, Minh

17. Bảng `loyalty_points` & 18. `loyalty_point_histories`

Lưu tổng điểm của Customer (`customer_id` PK/FK) và lịch sử cộng/trừ điểm.

19. Bảng `order_statuses` & 20. `orders` (Hỗ trợ State, Facade)

Tên cột Kiểu dữ liệu Ràng buộc
id INT PK AUTO_INCREMENT
customer_id INT FK NOT NULL
status_id INT FK NOT NULL
voucher_id INT FK NULL
total_amount DECIMAL(12,2) NOT NULL

21. Bảng `order_details`

Tên cột Kiểu dữ liệu Ràng buộc
id INT PK AUTO_INCREMENT
order_id INT FK NOT NULL
variant_id INT FK NOT NULL
quantity INT NOT NULL
subtotal DECIMAL(12,2) NOT NULL

22. Bảng `order_status_history` & 23. `payment_methods` & 24. `payments`

Lưu lịch sử đổi trạng thái đơn và phương thức/dữ liệu thanh toán của đơn hàng.

Nhóm Hóa đơn & Sự kiện Ly

25. Bảng `sales_invoices` & 26. `purchase_invoices` (Hỗ trợ Factory Method)

Tách biệt hóa đơn nhập/xuất để khởi tạo Object phù hợp. Liên kết với `customers` / `suppliers` và `users`.

27. Bảng `sales_invoice_details` & 28. `purchase_invoice_details`

Lưu chi tiết biến thể (`variant_id`), số lượng, đơn giá cho từng loại hóa đơn.

29. Bảng `events` & 30. `event_subscriptions` & 31. `notifications` (Hỗ trợ Observer)

Tên cột Kiểu dữ liệu Ràng buộc Mô tả
id INT PK AUTO_INCREMENT Bảng `events`
customer_id, event_id INT PKFK Bảng `event_subscriptions`
customer_id, event_id INT FK Bảng `notifications`

Hỗ trợ Design Pattern theo Database

Thành viên Design Pattern Bảng liên quan Mục đích & Cách hỗ trợ
Ly Factory Method sales_invoices, purchase_invoices Bóc tách 2 loại bảng hóa đơn giúp Factory chọn đúng loại object để khởi tạo.
Ly Observer events, event_subscriptions Khách hàng vào `event_subscriptions` là Subscriber. Tạo event sinh ra `notifications`.
Linh Abstract Factory products Trường product_type (Basic/Premium) giúp rẽ nhánh tạo đúng Family sản phẩm.
Linh Adapter products, product_variants... Dữ liệu chia nhỏ nhiều bảng, Adapter giúp gom các ResultSet JOIN thành 1 object Product.
Lương Builder customers, products Khởi tạo đối tượng nhiều cột an toàn trước khi Insert.
Lương Decorator promotions, product_promotions Khuyến mãi lưu ở bảng riêng (không nằm ở product). Decorator sẽ bọc đè promotion lên Product.
An Strategy vouchers Trường discount_type quyết định thuật toán (Percent, Fixed) tại runtime.
An State orders, order_statuses Trạng thái lấy từ DB (`status_id`) kích hoạt Class State tương ứng (Pending/Paid).
Minh Singleton users, user_sessions Load 1 lần lúc đăng nhập, duy trì phiên làm việc UserSession tĩnh.
Minh Facade Nhiều bảng Checkout Gom 7-8 thao tác (trừ kho, cộng điểm, đổi state...) lôi từ nhiều bảng vào chung 1 Transaction API.

Hướng dẫn chạy dự án

Dự án sử dụng Java thuần (không dùng Maven plugin để run trực tiếp) hoặc chạy qua file JAR. Mở thư mục gốc trong Terminal:


1. Biên dịch toàn bộ mã nguồn:

javac -d bin -sourcepath src/main/java src/main/java/com/clothingstore/Main.java

2. Chạy ứng dụng (Nhớ link JDBC driver):

java -cp "bin;lib/mysql-connector-j.jar" com.clothingstore.Main

Mẹo: Khuyến nghị mở thư mục BTL_DesignPattern_N2 bằng IntelliJ IDEA hoặc Eclipse, IDE sẽ tự động nhận diện JDK và thư viện JDBC để ấn nút Run một cách nhanh nhất.
